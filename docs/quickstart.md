# Quick Start

The quickest useful way to use ACM is not to set up everything. It is to get one small run working from end to end.

Start by signing in to the site and opening the app at `/app/`. Then open Settings and save one provider key for a service you already use. You do not need every provider. One working key is enough to begin.

After that, go to Presets or Configure. Pick a preset instead of building everything from scratch. A preset gives you a working starting point, which is much easier than trying to understand every setting at once.

Choose one document or one small piece of input. Keep it small on purpose. The point of your first run is to prove the system works and to see the shape of the workflow, not to run a giant experiment.

Then run one model through one workflow. Let the run finish. Open the Execute page while it is running, then open History when it is done.

If that worked, you have already done the most important thing. You have proven that your account, provider key, preset, and run flow all work together.

Only after that should you start making the run bigger. Add more models. Change the settings. Add evaluation. Add combine. Bring in GitHub if you need it. The first win should stay simple.

If you want to understand the words ACM uses while you do this, read [core-concepts.md](core-concepts.md). If you want to understand what the longer workflow does, read [full-pipeline.md](full-pipeline.md).
