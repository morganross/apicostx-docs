# Provider Keys and GitHub

The site needs two different kinds of connections, and it helps to keep them separate in your mind.

Provider keys are what let the site talk to outside AI or search services. If you want a run to actually do work, you usually need at least one provider key saved in Settings.

GitHub connections are different. They are for browsing or importing files from a repository. A GitHub connection does not replace a provider key, and a provider key does not replace GitHub access.

If you are new, do not connect everything at once. Save one provider key first. Get one small run working. Only then add GitHub if you need repository files.

If a provider key is missing or invalid, runs may fail before they really start.

If a GitHub connection is missing or invalid, imports or repository browsing may fail even if the rest of the site is working.

When in doubt:

Start with provider keys.
Use GitHub only when your input really needs to come from a repository.
