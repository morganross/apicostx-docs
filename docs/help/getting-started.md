# Getting Started

The best first use of the site is a small run that actually finishes.

Do not try to understand every setting on day one. Do not try to connect everything. Just get one simple run working from start to finish.

Start by signing in. Then open the app page. Go to Settings and save one provider key for a service you already use. One working provider key is enough to begin.

After that, go to Presets or Configure. Pick a preset instead of building a setup from scratch. A preset gives you a working starting point and keeps the first run simple.

Use one small document or one small piece of input. Then run one model. Watch it on the Execute page. When it finishes, open History and look at the saved result.

If that works, you have already done the most important thing. You have proven that your sign-in, provider key, preset, and run flow all work together.

After that first success, you can make things bigger. Add more models. Change settings. Add evaluation. Connect GitHub if you need to bring files in from a repository. But the first run should stay small on purpose.

If you are not sure what each page is for, read [main-pages.md](main-pages.md).
