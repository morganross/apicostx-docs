# Main Pages

The easiest way to understand the site is to understand what each page is for.

The Quality page is for looking at model quality and cost comparison data. It helps you think about model choices. It is not where you start a run.

The Presets page is where you see saved starting setups. If you are new, this is usually a good place to begin because it saves you from filling in every option yourself.

The Configure page is where you shape a run. This is where you choose or adjust the setup before starting the work.

The Execute page is where you watch a run while it is happening. If you want to know whether the work is still running, failed, paused, or finished, this is the page to watch.

The History page is where old runs live. If you want to reopen a result, compare past work, or inspect a failure after the fact, go here.

The Content Library is where reusable inputs live. That can include documents, instructions, and evaluation material.

The Settings page is where you manage provider keys and GitHub connections. If a run cannot start because something is missing, Settings is one of the first places to check.

If you need help with provider keys or GitHub connections, read [provider-keys-and-github.md](provider-keys-and-github.md).
