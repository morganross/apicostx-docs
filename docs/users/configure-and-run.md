# User Guide: Configure and Run

## Typical Flow

1. Open `Presets` or `Configure`.
2. Choose an existing preset, or start with a default configuration.
3. Review the generation, evaluation, and combine settings.
4. Confirm your document and content-library inputs.
5. Start the run.
6. Move to `Execute` to watch progress.
7. Use `History` to revisit finished runs.

## Configure Page

The Configure page is the main control surface for a run. It contains sections for:

- general run settings
- FPF settings
- GPTR settings
- DR settings
- multi-agent settings
- evaluation settings
- concurrency settings
- combine settings

Not every option needs to be changed every time. Most users should start from a preset and only adjust the sections relevant to the current run.

## During Execution

Use the Execute page to:

- see whether the run is pending, running, paused, completed, or failed
- inspect progress
- view logs
- move into detailed run results when they are ready

## After a Run

Use History to:

- reopen an old run
- compare completed work
- inspect failures
- clean up old runs if needed

## Good Habits

- Save or choose a preset before making large changes.
- Make sure provider keys are configured before starting a run.
- If a run fails early, check Settings and then retry rather than repeatedly launching the same broken configuration.
